# ask-me

在动手之前，先把不确定的事情问清楚。适用于方案清楚，但是怕 AI 理解偏差的场景。

> 还有姐妹版：[grill-me](https://ali-skills.alibaba-inc.com/skills/liaoyikang.liaoyik/skills/grill-me) 技能，可以用来对方案进行结构化、无情追问，逐一解决决策树每个分支，适用于和 AI 探讨方案。

## 简介

ask-me 是一个面向 AI 编程助手的 skill。它的作用只有一件事：**在 AI 开始写代码或给出方案之前，强制它先识别信息缺口，逐步向你追问，直到所有不确定点都解决**，再动手。

默认情况下，AI 倾向于立刻开工——用假设填补空白、用通用方案覆盖具体需求。
ask-me 打断这个默认行为，把"先问清楚"变成强制流程，减少返工次数，减少 Token 消耗。

## 为什么需要

AI 的常见问题不是能力不足，而是**在信息不足时仍然给出答案**。它会用合理的假设填补空白，代码写出来了，但方向可能从一开始就偏了。

几个典型场景：

- 你说「帮我做一个列表组件」，AI 用了本地 `useState` 管状态，但你的项目全局用 Zustand，后续集成要全部返工
- 你说「帮我写一个文件上传接口」，AI 用了内存存储（`multer` 默认），但你的服务是多实例部署，文件根本不能放本地
- 你说「帮我配一下 ESLint」，AI 装了 `eslint-config-airbnb`，但你的项目是 TypeScript + React 18，规则集完全不对

这类返工的根本原因不是 AI 写错了，而是**它在不该动手的时候就动手了**。

ask-me 的解法是：在实施之前，让 AI 走一遍信息扫描流程，主动暴露它无法从代码库或已有知识中自行解决的不确定点，每轮最多问 2 个问题，给出推荐答案和理由，等你确认后再继续。

## 触发词 / 如何启用

在你的消息中加入以下任意表达，AI 就会进入 ask-me 模式：

- `你需要问我什么`
- `先确认一下`
- `别急着写代码`
- `不清楚的地方先问`
- `有什么需要我提供的`
- `有不清楚的先问我`
- `/ask-me`（直接调用 skill）

任何明确要求 AI 先澄清再动手的表达都有效。

![2026-03-24 23.06.34](https://liaoyk-markdown.oss-cn-hangzhou.aliyuncs.com/markdownImg_2023/2026-03-24%2023.06.34.gif)

有 ask-me 的对话，AI 在开始写代码之前，先问了我两个问题：

1. 「刷新后数据来源是什么？」
2. 「下拉刷新的出发范围？」

我回答了之后，AI 才开始写代码。

虽然都实现了需求，但有 ask-me 的对话，AI 写出来的代码更符合我的预期，没有返工。

我这里例子只是一个使用效果的展示，实际使用中，ask-me 可以用在：

- 需求模糊、缺少关键上下文
- 有多种截然不同的实现方向
- 涉及外部系统或用户私有逻辑
- 任务目标不清晰
