Interactive Walkthrough · 部分退款 01 / 05

用户进入订单详情页

客户端 · 加载订单 + 计算可退项
动作
点击「我的订单」→ 选中已支付订单 → 查看详情
接口
GET /v2/order/{id} · 返回订单 + 商品行 + 可退项标记
校验
  • 订单状态必须是「已支付 / 已完成」
  • 每个商品行计算 refundable_amount
产物
退款入口按钮（不可退场景显示置灰 + 原因）

用户勾选要退的订单项

客户端 · 实时计算预算
动作
勾选机票（¥800）+ 保险（¥80），酒店未勾选
接口
POST /v2/refund/preview · body: { items: [...] }
校验
  • 单项金额 ≤ 商品支付金额
  • 总额 ≤ 订单实付（防多退）
  • 互斥规则：机票退则险种自动退
产物
预算金额 ¥880 + 退款规则提示

用户确认退款金额并提交

Refund Svc · 创建退款单
动作
查看汇总 → 选择退款通道（默认原路） → 提交
接口
POST /v2/refund · 创建退款单 → DRAFT → PENDING
校验
  • 幂等：同 idempotency-key 返回原退款单
  • 金额 > 500 元 → 路由二级审批
  • 金额 ≤ 500 元 → 跳过审批直接执行
产物
退款单 ID + 状态 PENDING / APPROVED

主管审批退款单

OA 系统 · 审批通过后异步回调
动作
张主管在 OA 收到审批通知，查看明细后通过
接口
OA → POST /v2/refund/{id}/approve 回调 Refund Svc
状态
退款单 PENDING → APPROVED，触发后台执行
风险
SLA 2 小时未审批 → 自动提醒；超过 24 小时未审批 → 自动驳回 + 通知用户

调用退款 & 通知对账

Refund Svc → Payment Adapter → 对账 MQ
调用
按子单串行调用 refund.fullRefund（复用现有接口）
状态
APPROVED → REFUNDED（全部子单成功） / PARTIAL_FAILED（部分失败 → 重试 3 次后人工介入）
下游
发 refund.completed 事件到 RocketMQ，对账系统消费 · 兼容现有 schema 无需改造
用户
退款成功通知（推送 / 短信），订单详情显示退款明细
